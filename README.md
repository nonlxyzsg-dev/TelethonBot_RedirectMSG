# TelethonBot_RedirectMSG

Telegram-бот, который автоматически пересылает сообщения (текст, фото, видео,
файлы, альбомы, ссылки) из одного чата в другой. Работает от вашего аккаунта
через Telethon. Есть поддержка MTProto-прокси для обхода блокировок.

---

## Быстрый старт (самый простой способ — EXE)

1. **Скачайте последний релиз**
   Перейдите на страницу [Releases](../../releases) и скачайте:
   - `TelethonBot-vX.Y.Z.exe` — сама программа
   - `.env.example` — шаблон настроек (переименуйте в `.env`)
   - `proxies.txt` — список прокси (можно не трогать, там уже есть примеры)

2. **Положите все три файла в одну папку**
   Например, создайте папку `C:\TelethonBot\` и положите туда EXE, `.env`
   и `proxies.txt`. Программа будет хранить всю свою работу только в этой
   папке — ничего не пишет в AppData, реестр или другие места. Можно
   переносить папку куда угодно (флешка, другой диск) — работает.

3. **Заполните `.env`**
   Откройте `.env` в Блокноте и впишите свои значения. Где взять — ниже.

4. **Запустите EXE двойным кликом**
   Откроется консоль. При первом запуске Telegram пришлёт код — введите
   его прямо в консоли. Больше пароль не спросит, пока файл `telethon.session`
   рядом с EXE.

5. **Не закрывайте консоль** — пока окно открыто, бот работает.
   Для остановки нажмите `Ctrl+C` в консоли (так правильнее всего).

## Где взять значения для `.env`

### API_ID и API_HASH
1. Зайдите на https://my.telegram.org, войдите своим номером.
2. Нажмите "API development tools".
3. Заполните любые "App title" и "Short name" (например, "MyBot" и "mybot").
4. После регистрации увидите `api_id` (число) и `api_hash` (строка).
   Скопируйте в `.env`.

### MONITORED_CHAT_ID (откуда пересылаем) и CHAT_ID_TO_REDIRECT_MESSAGES (куда)
1. Откройте в Telegram бота `@userinfobot` (или `@getmyid_bot`).
2. Напишите ему, он пришлёт ваш ID.
3. Чтобы узнать ID группы/канала: перешлите любое сообщение из него
   боту `@userinfobot` — он покажет ID источника.
4. Для супергрупп и каналов ID начинается с `-100…` — копируйте целиком,
   включая минус.
5. **Важно:** ваш аккаунт должен состоять в обоих чатах.

### Остальные поля
Можно не трогать. Менять `DEVICE_MODEL`, `SYSTEM_VERSION`, `APP_VERSION`
после первого логина **не рекомендуется** — Telegram считает это сменой
устройства и может уведомить вас или завершить другие ваши сессии.

## Как работают прокси (файл `proxies.txt`)

Telegram иногда блокируют провайдеры или страны. Если бот не может
подключиться напрямую, он сам переберёт прокси из `proxies.txt` и
использует первый работающий.

**Формат:** одна строка — один прокси, вид `tg://proxy?server=HOST&port=PORT&secret=HEX`.
Строки, начинающиеся с `#`, считаются комментариями.

**Пример:**
```
# Рабочие MTProto-прокси (обновляется редко, если все не работают — поищите свежие)
tg://proxy?server=194.87.57.180&port=443&secret=ddb7c4685b50c48b91a64412b63408dc2c
tg://proxy?server=194.87.57.180&port=8443&secret=ddb7c4685b50c48b91a64412b63408dc2c
```

**Где найти свежие прокси:** каналы `@MTProtoProxies`, `@mtpro_xy`, боты
`@mtproto_bot`, `@MTProxyBot`. Скопируйте ссылку прокси из Telegram
(при тапе по прокси есть "Поделиться") — она как раз в формате
`tg://proxy?...`.

**Если прокси вам не нужны** — можно оставить файл с примерами, он
не помешает: программа всё равно сначала пробует напрямую, и только
если не вышло — идёт по списку.

## Частые вопросы (FAQ)

### "Invalid api_id/api_hash"
Скопированы не те значения или с пробелами. Перепроверьте в .env.

### "Peer not found" / "Cannot find any entity"
Ваш аккаунт не состоит в чате. Добавьтесь в чат вручную, отправьте
туда любое сообщение (чтобы чат появился в диалогах), перезапустите бот.

### Меня выкинуло из Telegram после запуска
Обычно это значит, что изменился `DEVICE_MODEL` / `SYSTEM_VERSION` /
`APP_VERSION` между запусками, или пропал файл `telethon.session`.
Решение: не меняйте эти поля в `.env`, и храните `.session`-файл
рядом с EXE. Если потеряли — заново залогиньтесь, это безопасно.

### "Flood wait"
Telegram попросил подождать N секунд — это нормально, бот ждёт и
продолжает. Если падает каждый раз — уменьшите интенсивность
(например, перешлите вручную старые сообщения, не автоматом).

### Не работает прямое подключение
Добавьте свежие MTProto-прокси в `proxies.txt` и перезапустите.

## Запуск из исходников (для разработчиков)

```bash
# Клон и установка
git clone <url>
cd TelethonBot_RedirectMSG
pip install -r requirements-dev.txt

# Запуск
python -m bot
# или
python main.py

# Тесты и линт
ruff check src tests
pytest -q

# Ручная сборка EXE (если есть Windows)
pyinstaller bot.spec
# результат — в dist/
```

Подробности архитектуры и конвенции коммитов — в `CLAUDE.md`.

## Для старых пользователей (апдейт с предыдущей версии)

Было: `config.json` рядом со скриптом, хранил api_id/api_hash и прочее.

Стало: `.env` в том же месте, формат `KEY=VALUE`. **Миграция автоматическая:**
при первом запуске новой версии бот сам переносит значения из `config.json`
в `.env`, а старый json сохраняет как `config.json.bak` (чтобы ничего
не потерялось). Никаких действий от вас не требуется.

Аналогично с сессией: если у вас есть `telethon.session` в папке, откуда
вы раньше запускали скрипт — положите его рядом с новой программой (EXE
или main.py), бот его подхватит и не попросит логиниться заново.
